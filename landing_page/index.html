How It Works

A sophisticated approach to security

1

Process Scanning

Periodically scans all running processes and their command line arguments.

2

Pattern Matching

Applies regex-based detection to identify commonly used reverse shell patterns.

3

Network Analysis

Examines suspicious network connections that may indicate command and control channels.

4

LLM Verification

Uses Google Gemini to analyze suspicious processes and reduce false positives.

5

Threat Response

Terminates malicious processes and sends detailed email notifications to admins.

Installation

Quick and easy setup process

Install from Source

Clone the repository and install using pip:

# Clone the repository
git clone https://github.com/hamidriaz1998/ReverseShellKiller.git
cd ReverseShellKiller

# Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate

# Install in development mode
pip install -e .

Using UV Package Manager

# Clone the repository
git clone https://github.com/hamidriaz1998/ReverseShellKiller.git
cd ReverseShellKiller

# Create a virtual environment with UV
uv init

# Install packages
uv sync

Remember to set up your environment variables in a .env file as described in the Configuration section before running the tool.

Building a Linux ELF Binary

To build a standalone executable:

# Make sure you're in a virtual environment with dependencies installed
uv add pyinstaller

# Build the executable
pyinstaller --clean linux_build.spec

# Executable will be available at dist/reverse-shell-killer

Running as a System Service

For persistent protection, you can set up Reverse Shell Killer as a systemd service. Make sure you've set up the required environment variables in a .env file first (see Configuration):

Option 1: Using the Installation Script (Recommended)

  1. Build the binary: Follow the steps in "Building a Linux ELF Binary"
  2. Create the .env file: Set up your credentials in a .env file in the project root
  3. Run the installation script:
# Make the script executable (if needed)
chmod +x scripts/install_service.sh

# Run the installation script from the project root directory
# Example: Install with default settings (interval 10, default log, no LLM)
sudo ./scripts/install_service.sh

# Example: Install with LLM enabled and a 5-second interval
sudo ./scripts/install_service.sh --use-llm --interval 5

# Example: Install with a custom log file
sudo ./scripts/install_service.sh --logfile /var/log/custom_revshell.log

Configuration

Customize to fit your security needs

Command-Line Arguments

Reverse Shell Killer supports various command-line arguments to customize its behavior:

--interval

Scan interval in seconds (default: 10)

Example: --interval 5 to scan every 5 seconds

--logfile

Path to log file (default: ./revlogs.log)

Example: --logfile /var/log/revshell.log

--dry-run

Detect suspicious processes but do not terminate them

Useful for testing before deploying in production

--metrics

Enable Prometheus metrics collection

Exposes metrics on port 8000 for monitoring

--use-llm

Enable LLM-based analysis for improved detection accuracy

Requires Google Gemini API key

Example Usage

# Basic usage with default settings
reverse-shell-killer

# Advanced configuration
reverse-shell-killer --interval 5 --logfile /var/log/revshell.log --use-llm --metrics

Environment Variables

Reverse Shell Killer uses environment variables for sensitive configurations like API keys and email credentials. These should be stored in a .env file in the project's root directory.

Sample .env File

# Google Gemini API Key for LLM-based detection
GEMINI_API_KEY=your_gemini_api_key_here

# Email notification settings
EMAIL_USER=your_email@gmail.com
EMAIL_PASS=your_gmail_app_password
EMAIL_RECIPIENT=recipient_email@example.com

Available Environment Variables

GEMINI_API_KEY

Your Google Gemini API key for LLM-based detection

Required when using --use-llm flag

Get your API key from the Google AI Studio

EMAIL_USER

Gmail address to send notifications from

Required for email notifications

EMAIL_PASS

Gmail app password (not your regular Gmail password)

Required for email notifications

Generate an App Password in your Google Account Security settings

Security Note

Keep your .env file secure and never commit it to version control. The file is included in .gitignore by default.

Frequently Asked Questions

Common questions about Reverse Shell Killer

What is a reverse shell?

A reverse shell is a type of shell in which the target machine initiates a connection back to the attacker's machine. This is commonly used in cyberattacks to bypass firewalls and gain unauthorized access to systems.

Does this tool require root privileges?

Yes, to monitor all processes and terminate malicious ones, this tool needs to run with root privileges. You can run it using sudo or as the root user.

Can I use this on production servers?

Yes, but we recommend running it in dry-run mode (--dry-run) first to ensure it doesn't terminate any legitimate processes in your environment. Once you're confident in its detection capabilities, you can enable automatic termination.

How do I set up email notifications?

Create a .env file in the same directory as the executable with the following variables:

EMAIL_USER=your-email@gmail.com
EMAIL_PASS=your-app-password
EMAIL_RECIPIENT=admin@yourcompany.com

Note: EMAIL_PASS should be a Gmail App Password, not your regular password. You can generate one in your Google Account security settings.

How can I enable LLM-powered detection?

For enhanced detection accuracy, the tool can use Google's Gemini API:

  1. Get a Gemini API key from the Google AI Studio
  2. Add your API key to the .env file as GEMINI_API_KEY=your-api-key
  3. Run with the --use-llm flag: reverse-shell-killer --use-llm
